5. Representation of a chapter¶

Abstraction for one “chapter”, i.e., one of the EPUB instances that together constitute the final book.

class rp2book.chapter.Chapter(directory_name)[source]¶

Class for a chapter, providing all the information that is necessary for the creation of “top level” administration files like tables of content or package files.

Parameters:directory_name (string) – the directory in the file system containing the chapter. All information will be extracted with this as a base.
creators¶

Chapter creator

date¶

Chapter title

nav¶

Abstraction for the information in the package file; an instance of NAV.

ncx¶

Abstraction for the information in the package file; an array of ElementTree.Element objects for the table of content.

opf¶

The root of the navigation, an ElementTree.Element object

source¶

Directory name pointing at the chapter.

target¶

Target directory for the copied chapter

title¶

Chapter title

class rp2book.chapter.NCX(chapter)[source]¶

Abstraction of the old type TOC file, providing the run-time information important for processing. Its expected name and position within the chapter is toc.ncx.

toc¶

An array of an ElementTree.Element objects

class rp2book.chapter.Nav(chapter)[source]¶

Abstraction of the new type navigation (TOC) file, providing the run-time information important for processing. Its expected name and position within the chapter is nav.xhtml.

nav¶

The navigation root element itself, an ElementTree.Element object

class rp2book.chapter.OPF(chapter)[source]¶

Abstraction of the package file, providing the run-time information important for processing. Its expected name and position within the chapter is package.opf.

static _get_manifest(dir_name, root)[source]¶

Collect the manifest entries, converting them on the fly, namely:

  • change the @href attributes to include the directory name
  • change the @id attributes to include the directory name (normalized to ‘-‘ instead of ‘/’)
  • removing entries for navigation
Parameters:
  • dir_name (string) – the directory in the file system containing the chapter. All information will be extracted with this as a base.
  • root (ElementTree.Element) – the Root element of the package file
Returns:

a list of ElementTree.Element objects
